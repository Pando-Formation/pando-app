PANDO · Design System

Foundations for everything
we build.

A single source of truth for PANDO's visual language. Built on a layered token architecture, a precise type scale, and Apple-grade attention to rhythm, contrast, and restraint. Every value here is a decision, not a default.

01 — Foundations

Principles

Five rules that govern every decision in this system. When in doubt, return here.

Clarity first
Content is the interface. Typography and spacing carry hierarchy — not decoration. Remove anything that competes with meaning.
Whitespace is structure
Generous, consistent spacing groups what belongs together and separates what doesn't. Space is an active design tool, never filler.
Color means action
Blue is reserved for what's interactive. When everything is colored, nothing reads as clickable. Restraint trains the eye.
Depth with restraint
Elevation and translucency convey relationships between layers. Used sparingly, they orient. Overused, they create noise.
Consistency through tokens
No magic numbers. Every color, size, and space references a token. Change the token, change the system — everywhere, at once.
02 — Foundations

Token architecture

Three layers, following the W3C Design Tokens standard. Components only ever reference the top layer — never raw values. This is what lets the entire system re-theme by changing a handful of mappings.

Layer 1
Primitive tokens

Raw, context-free values. The complete palette of what's possible. Never used directly in a component.

--blue-500: #3b82f6 · --space-5: 16px · --size-title-1: 34px
↓
Layer 2
Semantic tokens

Map primitives to roles by intent. This is where theming lives — light/dark just remap these.

--color-accent: var(--blue-500) · --color-text-primary · --color-bg-base
↓
Layer 3
Component tokens

Per-component values that reference semantic tokens. The only layer components read from.

--button-radius: var(--radius-3) · --card-bg: var(--color-surface)

Responsive scale

Three breakpoints, following Apple's exact system. Type doesn't scale fluidly — it steps down at each breakpoint, and letter-spacing loosens as size shrinks so small text keeps its air. Resize this window to watch the specimens above respond live.

Body and UI text stay fixed across breakpoints — only display and title sizes step down. This keeps reading text legible while letting headlines breathe on large screens and tighten on small ones.

The Apple letter-spacing rule, applied exactly: tracking and size move in opposite directions. At 96px, display-1 uses −0.015em (tight). At 48px on phone, it relaxes to −0.003em. Small text gets positive tracking for air. This inverse relationship — not the sizes themselves — is what makes type feel professionally set rather than merely scaled.

06 — Foundations

Section rhythm

Component spacing tops out at 80px. Page sections need far more room to breathe — this is what makes Apple's pages feel calm and premium rather than cramped. A separate scale governs the vertical gaps between major sections, and it steps down responsively.

SECTION SPACING SCALE — desktop → phone
TokenDesktopTabletPhoneUse
section-sm80px64px48pxBetween subsections
section-md120px96px72pxDefault between sections
section-lg160px128px96pxMajor section breaks
section-xl200px160px120pxHero / dramatic pauses
Rule of thumb: if two blocks belong to the same idea, use component spacing (≤80px). If they're separate ideas the reader should mentally file apart, use a section token. The gap itself communicates structure.

Stats & figures

Big standalone numbers are how both Apple and PANDO communicate impact — "8x," "33 hours," or for us "211 000 €," "12 sessions," "84%." The pattern is always the same: an oversized tight-tracked number paired with a small uppercase label. Numbers use tabular figures so they align in columns.

Materials

Translucent surfaces with backdrop blur — for sticky nav bars, toolbars, and overlays that float over scrolling content. The blur lets the layer beneath show through softly, so the reader keeps their sense of place. Three thicknesses for different contexts.

content behind the material

Use sparingly. Materials are for elements that genuinely float over other content — a sticky top-bar, a command palette, a modal scrim. A regular card on a static background should use a solid surface, not a material.

Motion

Motion confirms actions and guides attention — it never decorates. Three durations, three easing curves. Hover any chip to feel the spring.

fast · 120ms

Easing: ease-out (0.16, 1, 0.3, 1) for entrances · ease-in-out for moves · ease-spring for playful, tactile feedback.
11 — Foundations

Composition

The recurring rhythm that opens almost every Apple section — and should open ours. Eyebrow → headline → intro, with defined spacing between each. Memorizing this one pattern means every page already has a consistent, premium opening.
